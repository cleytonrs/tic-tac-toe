# Tic Tac Toe

🕹️ A self-contained, browser-based Tic Tac Toe game. No external frameworks, no network dependencies — just open `index.html` in your browser.

## How to Play

1. Open `index.html` directly in your browser (file:// or any local server)
2. Customize player names (optional)
3. Click "Start Game"
4. Take turns clicking cells on the 3x3 board
5. After each round, choose "Play Again" or "Finish Game"
6. On the final screen, share your results on Twitter/Facebook

## Features

- 3x3 board with win and draw detection
- Customizable player names (1–20 characters)
- Persistent scoreboard across rounds
- Starting player alternates each round
- Social media sharing (Twitter/Facebook) with clipboard fallback
- Responsive design (320px–1920px)
- Works 100% offline with no external dependencies

## Development

### Prerequisites

- Node.js 18+

### Setup

```bash
npm install
```

### Build

Compiles `game.ts` to `game.js` (browser-ready, no module exports):

```bash
npm run build
```

### Tests

```bash
npm test
```

Tests include:

- **Property-based tests** (fast-check): universal correctness properties for game logic
- **Unit tests**: name validation, score management, and HTML DOM structure

### Project Structure

```
├── index.html          # HTML structure (markup only)
├── styles.css          # Stylesheet (layout, colors, transitions, responsive)
├── app.js              # Application controller (DOM wiring, event handlers, state)
├── game.ts             # TypeScript source (pure game logic)
├── game.js             # Compiled JavaScript (generated by build)
├── package.json        # Dev dependencies
├── tsconfig.json       # TypeScript configuration
├── vitest.config.ts    # Test configuration
└── tests/
    ├── game-logic.property.test.ts   # Property-based tests
    ├── game-ui.test.ts               # DOM structure tests
    └── name-validation.test.ts       # Name validation tests
```

### Architecture

The project follows a clear separation of concerns:

| File | Responsibility |
|------|---------------|
| `index.html` | Structure — semantic HTML for all four screens |
| `styles.css` | Presentation — layout, colors, transitions, responsive design |
| `game.js` | Logic — pure functions for board evaluation, scoring, validation |
| `app.js` | Behavior — DOM queries, event binding, state management, screen transitions |

## Technologies

- TypeScript (compiled to vanilla JS)
- HTML5 + CSS3 (separate files)
- Vitest + fast-check (testing)
- JSDOM (DOM test environment)
